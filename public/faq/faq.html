What is SpotDB?

SpotDB is a platform that lets you search for all items on Spotify. SpotDB is completely open-source platform that uses the official Spotify WebAPI, as well as additional third-party APIs to gather information like lyrics, producers, writers and more.

Where do we find this data?

SpotDB uses a few APIs to complete certain tasks, such as search, fetch data, gather lyrics and more. Mainly, we use Spotify's WebAPI that gives us all needed information.

  • Spotify WebAPI - here - for searching and fetching data straight from Spotify
  • Genius API - here - for gathering track lyrics, producers and songwriters
  • (Replaced) lyrics.ovh API - here - was previously used for gathering track lyrics, now replaced


    • Even though we use the official Spotify API, there still are limitations. Find out more here.

    Can I contribute to SpotDB?

    Yes, SpotDB is open-source. You can contribute by visiting our GitHub repository, creating Issues and more. Extremely grateful for all help :)

    Where can I get the source code?

    The source code is available in our GitHub repository!

    What language is SpotDB written in?

    SpotDB is a vanilla HTML/CSS/Javascript website together with Node.js + Express backend.

    Does SpotDB have an API?

    No, we directly fetch Spotify data and if you need that data, you should do that too. The Spotify WebAPI is completely free and easy to get started with, and you will find it here.

    We are an open-source and free-to-use website made by a single Spotify user. We can't just create APIs out of nowhere and maintain them.

    Why are there no charts, streams, etc.?

    SpotDB directly fetches data straight from Spotify and currently, the Spotify WebAPI does not provide such data. In the long term, we may start using third-party and/or external APIs to get such data, if needed.

    What can I search?

    You can pretty much search for everything on Spotify, with one exception.

    Albums, tracks (+ EPs), artists, playlists, episodes and shows. To see how to find Spotify users on SpotDB, click here.

    What data is missing?

    Even though we are using the official Spotify WebAPI, there are a few stuff that may be missing, or replaced with different APIs.

  • Lyrics - the Spotify API does not provide track lyrics, so we rely on the Genius API
  • Credits - the Spotify API does not provide credits (such as producers, songwriters), so we rely on the Genius API
  • Users - now, by going to the Users page, you can fetch user data from Spotify by putting in a Spotify UserID or a Display Name that is unique, if you're lucky. However, you can't Search for users, like you can with with artists, albums and more.

    • ID

    Each item in Spotify has a unique ID, that looks something like "7mzrIsaAjnXihW3InKjlC3" (a real ID). There cannot be two items with the same ID.

    Copyrights

    Copyrights are labels defined by the item's record label, artist or publisher, and represent who owns this item, legally speaking. There's more to this information, such as artists owning the songwriting rights, and more.

    Region (Market)

    Most items include "available_markets", which defines where this item can be found. There are multiple releases that are region-locked and can only be found there. This is defined by the item's record label, artist or publisher.

    External IDs

    Most items include "external_ids", which are IDs completely unrelated to the Spotify unique item ID. Examples of these external_ids can be the UPC or ISBN IDs.

    Embed

    You can embed Spotify albums, tracks, playlists, artists, episodes and shows, using an iframe embed. On all items excluding users we provide an "Embed" tab that lets you generate these embeds, preview them, and copy the code. You can customize the width and height and use them in your websites, HTML projects, and more.

    The lyrics are wrong!

    You're probably right. Once again, we use the Genius API to fetch lyrics, songwriters and producers info. Therefore, if there is an issue with lyrics, it is most likely due to Genius. However, if the lyrics are in a completely different language or don't correspond to the track entirely, it is on our side. The Genius API does not directly let you get a track's lyrics, so we use JSDOM to fetch it from the page itself. This could cause several issues, such as some songs being fetched in a different language.

    We apologize if you find these issues, and please contact us if you do, so we can work to resolve these issues. Thank you!

    You leaked this!

    Oh well. Once again, we fetch details from the Spotify WebAPI, and if something has been leaked, we are not the ones involved. We control NOTHING, and it is purely the fault of either the Spotify WebAPI or the labels/artists/publishers releasing data wrong or too soon. We are a public platform to fetch data, not intended to find private data...

    Do I have to sign in to use SpotDB?

    No, the way SpotDB works is... when you open SpotDB, a Spotify API access token is generated, which is an anonymous token completely defined for you, under the SpotDB API app. REMINDER, we do not see your tokens and data! With it, you can search, fetch data, and more. Completely, without having to sign-in. In fact, we don't even have Spotify sign-in's. Lol.
